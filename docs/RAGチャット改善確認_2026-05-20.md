# RAGチャット改善確認 2026-05-20

確認日時: 2026-05-20 12:01:24  
対象: `http://210.165.3.139/web/chat`  
方法: PlaywrightでRAGチャット画面から質問を送信し、画面表示とXHR応答の回答本文・参考資料を回収した。

## 結論

改善が進んだという前提で回帰確認したが、現時点では「誤答を避けるために回答を抑制している」ような挙動が強く、実務上は改善済みとは言いにくい。

4月時点は、資料に当たっても数値や表項目を誤って回答するケースが目立っていた。今回の確認では、正しい資料に当たるケースが一部ある一方、本文がほぼ `すみません、わかりません。` へ落ちるケースが多い。誤答リスクは下がった可能性があるが、正しい資料から回答を抽出できないため、設計者の調査用途としては後退している。

特に問題が残る点は以下。

- 正しい資料に当たっても、表・Excel・PDF本文から値や項目を抽出できない。
- `わかりません` と答えながら、無関係な参考資料を付ける。
- 追撃で条件を足すと、初回より根拠が悪化する。
- 出力形式指定により、直前条件と関係の薄い資料へ再着地する。

## 良くなった可能性がある点

- 一部の質問では、以前より安易に断定しない挙動になっている。
- T14では、4月時点にあったコマツ小山への客先混線は弱まり、少なくとも広島アルミ側資料に寄った。
- T20やT16のように、検索・ランキング段階では正しい資料を拾えているケースがある。

ただし、これは「資料抽出や回答生成の改善」ではなく、現時点では「回答抑制が強くなった」可能性が高い。

## 開発先へ確認したいこと

1. 直近改善で、回答生成のしきい値や `わかりません` 判定条件を変更したか。
2. 正しい資料に当たった場合でも本文を返さない原因は、表抽出、スニペット抽出、LLMプロンプト、後段ガードのどこにあるか。
3. `わかりません` の場合は、無関係な参考資料を表示しない制御にできるか。
4. 追撃時に、初回回答の根拠集合を再利用・再評価しているか。それとも毎回独立検索しているか。
5. 出力形式指定時に、条件本文より形式指定が優先されて検索条件が崩れていないか。
6. Excel/Word/PDFそれぞれで、回答生成に使う本文・表・セル範囲をどの単位でチャンク化しているか。

## 次の評価観点

- 正しい資料に当たった場合、本文で必要値を返せるか。
- `わかりません` の場合、参考資料欄を空にできるか。
- 追撃で条件を足したとき、初回より根拠が悪くならないか。
- Excel表のセル値、Word本文の数値、PDF規格表の列挙項目を、資料名・ページ/シート付きで返せるか。
